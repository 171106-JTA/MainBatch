Java Servlets

What are servlets?

Servlets are programs that runs on a web/application server.
web server vs application server?
Quick summary, web server provides contents to be viewed in web browser (via http) and app server provides things(business logic and process)to applications (Also have more functionality like version control and console).
Servlet acts as a middle layer between the client and the database(?)

Cookie Handling

What are cookies? aka HTTP Cookie, web cookie, Internet cookie, browser cookie, or simply cookie
Basically, cookies are files that are left on your computer after visiting a website. When you visit that website again, these cookies can be scanned by that website to give it information about your previous encounter of the website, usually at least.
Example of Creating cookie
Cookie cookie = new Cookie("email", request.getParameter("email"));
cookie.setMaxAge(60); // in seconds
response.addCookie(cookie);
response.setContentType("text/html");

Reading Cookie with Servlet

To create cookie call javax.servlet.http.Cookie Object by calling getCookie() method of HttpServerRequest. Then cycle through an array and use getName() and get Value() methods to access each Cookies and associated values


Cookie cookie = null;
Cookie[] cookies = null;
cookies = request.getCookies();
response.setContentType("text/html");
if( cookies != null ) {
out.println("Found Cookies Name and Value");

for (int i = 0; i < cookies.length; i++) {
cookie = cookies[i];
out.print("Name : " + cookie.getName( ) + ", ");
out.print("Value: " + cookie.getValue( ) + "
");
}
} else {
out.println("No cookies founds");
}

Servlet context vs config
ServletConfig parameters are specified for a particular servlet and are unknown to other servlets. It is used for initialising purposes.
ServletContext parameters are specified for an entire application outside of any particular servlet
