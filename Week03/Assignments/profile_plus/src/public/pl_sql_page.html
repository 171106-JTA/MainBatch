What are triggers?

Triggers are procedures stored in PL/SQL or Java that run (fire) implicitly whenever a table or view is modified or when some user actions or database system actions occur. Most commonly used triggers are triggers set for DML(Data Manipulation Language).

Transaction Isolation Level

ANSI/ISO SQL standard defines four levels of transaction isolation, with different possible outcomes for the same transaction scenario. These levels are defined in terms of three phenomena that are either permitted or not at a given isolation level.
These phenomena are ...

Dirty Read

You are permitted to read uncommited, or dirty, data.

Nonrepeatable Read

If read a tow at time t1 and try to reread that row at time t2, the row may have changed. It may have have been disappeared, it may have been updated etc.
A non-repeatable read occurs, when during the course of a transaction, a row is retrieved twice and the values within the row differ between reads.

Phantom Read

If you execute a query at time t1 and again at time t2, additional row may have been added to the data base which may affect your results. A phantom read occurs when, in the course of a transaction, two identical queries are executed, and the collection of rows returned by the second query is different from the first.
This differs from a nonrepeatable read in that with a phantom read, data you already read hasn't been changed, but instead more data satisfies your query criteria than before.

Example of Phantom read vs non-repeatable read

  • User A runs the same query twice.
  • In between, User B runs a transaction and commits.
  • Non-repeatable read: The A row that user A has queried has adifferent value if you queries second time.
  • Phantom read: All the rows in the query have the same value before and after

On to actual isolation levels.

There are 4 isolation levels highest to lowest, they are Serializable, Repeatable read, read commited and read uncommunited.

Isolation Level Dirty read Non-repeatable read Phantom read
Read Uncommited Permitted Permitted Permitted
Read Commited Not permitted Permitted Permitted
Repeatable Read Not permitted Not permitted Permitted
Serializable Not permitted Not permitted Not permitted

Oracle Data base EXPLICITLY supports the READ COMMITED and SERIALIZABLE isolation level as they're defined in the standard.
However, REPEATABLE READ is the isolation level that the SQL standard claims will guarantee a read-consistent result from SQL.
