SQL Tutorial

SQL Stands for Structured query language. It is used for manipulating database. Its keywords are seperated into 5 categories.

  • DDL, Data Definition Language
  • DML, Data Manipulation Language
  • DQL, Data Query Language
  • DCL, Data Control Language
  • TCL, Transaction Control Language

DDL Consists of CREATE, ALTER and DROP, and they go along with table_name.
DML consists of INSERT, SELECT, DELETE, UPDATE.
DQL consists of SELECT ... basically SELECT statement. It's not even mentioned most of the times.
DCL consists of GRANT/REVOKE, its about security
consists of SET_TRANSACTION, SAVEPOINT, COMMIT, ROLLBACK

For example of their usage go look at examples at https://www.w3schools.com/sql/default.asp

WHERE VS HAVING

HAVING works on aggregated data while WHERE clause works on row's data. Think of WHERE as pre filter and HAVING as post filter.

Fun Fact: HAVING was added to SQL because WHERE could not work with aggregate fucntion

Union vs JOIN

UNION puts lines from queries after each other, while JOIN makes a cartesian product and subsets it -- completely different operations.

UNION is written like
SELECT name, address, city FROM CUSTOMER
UNION
SELECT name, address, city FROM MARKET
where there needs to be same number of columns and same type of data whereas JOIN just needs one commom column and
SELECT * FROM Customers
INNER JOIN
Suppliers on Customers.city = Suppliers.city;
